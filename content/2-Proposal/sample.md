## Team Scope and Responsibilities

This project was developed by a team of five members. The team worked together to build an Enterprise Asset Management system, including the frontend interface, backend API, database design, deployment architecture, testing, and documentation.

| Member | Main Responsibility | Detailed Scope |
| --- | --- | --- |
| Dunũ | Frontend Admin Portal | Build the Admin Portal UI, including dashboard, sidebar/navigation, asset management pages, employee and department pages, category management, assignment, maintenance, inventory, reports, tables, forms, modals, loading states, dark/light mode support, and responsive layout. |
| Member A | Backend Core, Authentication, and Database Foundation | Build the Node.js/Express backend foundation, design the Prisma/MySQL schema, implement JWT authentication, role-based access control, user account status handling, password hashing, and backend health check. |
| Member B | Asset Management and Business Workflows | Implement asset categories, asset CRUD, assignment/return/transfer workflows, maintenance/support request handling, inventory sessions, reporting logic, and related business rules for the asset lifecycle. |
| Member C | AWS Architecture, Deployment, and Workshop Documentation | Analyze the full-stack system architecture, design the AWS deployment model, prepare the deployment guide, configure the deployment flow with Amplify, ALB, Elastic Beanstalk and RDS, document testing and clean-up steps, and support converting the project into a technical workshop. |
| Member D | Frontend Employee Portal, QA, and User Experience | Build the Employee Portal UI, including employee dashboard, my assets, asset detail, support requests, FAQ, profile, change password, history pages, bilingual UI review, UI smoke testing, demo script, and user-facing feedback improvements. |
