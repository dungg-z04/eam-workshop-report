Blogs Posted

This section summarizes the technical blog posts we reviewed and shared during the internship program. The selected articles focus on Amazon EKS, Kubernetes operations, service mesh security, private networking, and AI-assisted cloud security on AWS.

Blog 1 - Amazon EKS now supports control plane egress through your VPC

Source: AWS Containers Blog - Amazon EKS now supports control plane egress through your VPC

This post introduces customer-routed control plane egress for Amazon EKS. It allows outbound traffic initiated by the Kubernetes API Server to be routed through the user’s VPC, improving network control, auditing, and compliance.

Blog 2 - Better Together: Amazon EKS Auto Mode and Istio Ambient Mesh

Source: AWS Containers Blog - Better Together: Amazon EKS Auto Mode and Istio Ambient Mesh

This post explains how Amazon EKS Auto Mode and Istio Ambient Mesh can be combined to reduce Kubernetes operational overhead while improving service-to-service security. EKS Auto Mode automates compute-layer tasks, while Istio Ambient Mesh provides sidecarless service mesh capabilities.

Blog 3 - Introducing AWS Continuum: Security at machine speed

Source: AWS Security Blog - Introducing AWS Continuum: Security at machine speed

This post introduces AWS Continuum, a new approach that helps security teams handle code vulnerabilities faster with AI. The content focuses on the vulnerability lifecycle: discovery, prioritization, validation, mitigation, and remediation.

I chose these topics because they are closely related to cloud-native operations, secure Kubernetes networking, modern infrastructure governance, and the Agentic AI trend in cloud security on AWS.
